Free login roadmap

Start with manual access, add real login later.

Google Sheet can list registered users and reference IDs. Private dashboards, OTP, and passwords need a real account system later.

01User submits enquiry and receives email PDF reference ID.
02User enters mobile number and reference ID on this page.
03User contacts WhatsApp support for course, payment, and class updates.
04Later, OTP/password login can open their private dashboard automatically.

Paid OTP later

Real OTP is optional for the future, not required now.

This static website and Google Sheet cannot securely send and verify mobile OTP by themselves. For a fully automatic login later, connect a provider such as Firebase Phone Auth, MSG91, Fast2SMS, Twilio, or WhatsApp Business API through a backend.

01Backend checks whether the mobile number exists in registered users.
02OTP provider sends SMS or WhatsApp OTP to that mobile number.
03User enters OTP; backend verifies it.
04Verified user sees only their own dashboard data.
